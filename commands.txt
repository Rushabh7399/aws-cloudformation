Perfect — here’s a **complete, practical command set** for working with
👉 **AWS CloudFormation** using the CLI.

I’ll keep it structured so you can use it like a cheat sheet during hands-on.

---

# 🚀 0. (Important) Use correct credentials

If using Assume Role:

```bash
aws sts get-caller-identity
```

👉 Must show:

```text
assumed-role/...
```

---

# 📄 1. Validate template

```bash
aws cloudformation validate-template \
  --template-body file://template.yaml
```

---

# 🚀 2. Create stack

```bash
aws cloudformation create-stack \
  --stack-name my-stack \
  --template-body file://template.yaml \
  --region ap-south-1
```

---

## ⚠️ If using IAM resources

```bash
--capabilities CAPABILITY_NAMED_IAM
```

---

# ⏳ 3. Check stack status

```bash
aws cloudformation describe-stacks \
  --stack-name my-stack \
  --query "Stacks[0].StackStatus"
```

---

# 📋 4. List all stacks

```bash
aws cloudformation list-stacks \
  --stack-status-filter CREATE_COMPLETE UPDATE_COMPLETE
```

---

# 📦 5. List resources inside stack

```bash
aws cloudformation list-stack-resources \
  --stack-name my-stack
```

---

# 🔍 6. Describe full stack details

```bash
aws cloudformation describe-stacks \
  --stack-name my-stack
```

---

# 📤 7. Get stack outputs

```bash
aws cloudformation describe-stacks \
  --stack-name my-stack \
  --query "Stacks[0].Outputs"
```

---

# 🔄 8. Update stack

```bash
aws cloudformation update-stack \
  --stack-name my-stack \
  --template-body file://template.yaml \
  --capabilities CAPABILITY_NAMED_IAM
```

---

# 🚀 9. Best command (create + update together)

👉 Use this instead of create/update separately:

```bash
aws cloudformation deploy \
  --template-file template.yaml \
  --stack-name my-stack \
  --region ap-south-1 \
  --capabilities CAPABILITY_NAMED_IAM
```

---

# 📊 10. Watch stack events (VERY useful)

```bash
aws cloudformation describe-stack-events \
  --stack-name my-stack
```

---

## 🔥 Live watch (debugging)

```bash
watch -n 5 aws cloudformation describe-stack-events --stack-name my-stack
```

---

# 🧹 11. Delete stack

```bash
aws cloudformation delete-stack \
  --stack-name my-stack
```

---

# ⏳ 12. Wait for completion (important)

```bash
aws cloudformation wait stack-create-complete \
  --stack-name my-stack
```

---

# 📁 13. Create stack from S3 (for large templates)

```bash
aws cloudformation create-stack \
  --stack-name my-stack \
  --template-url https://s3.amazonaws.com/bucket/template.yaml
```

---

# 🧪 14. Dry run (Change Set)

```bash
aws cloudformation create-change-set \
  --stack-name my-stack \
  --change-set-name my-change \
  --template-body file://template.yaml
```

---

# 🔍 15. Describe change set

```bash
aws cloudformation describe-change-set \
  --change-set-name my-change \
  --stack-name my-stack
```

---

# 🎯 Minimal daily workflow (recommended)

```bash
# Validate
aws cloudformation validate-template --template-body file://template.yaml

# Deploy (create/update)
aws cloudformation deploy \
  --template-file template.yaml \
  --stack-name my-stack \
  --capabilities CAPABILITY_NAMED_IAM

# Check status
aws cloudformation describe-stacks --stack-name my-stack
```

---

# ⚠️ Common errors (quick fixes)

| Error                   | Fix               |
| ----------------------- | ----------------- |
| AccessDenied            | Check IAM role    |
| Template error          | Validate template |
| Resource already exists | Use unique names  |
| Stack stuck             | Check events      |

---

# 💡 Pro tips

* Always use `deploy` instead of `create-stack`
* Always check **stack events** for debugging
* Always delete stacks to avoid charges
